terraform-gcp-iam

Comprehensive IAM Management for Google Cloud

Google Cloud Platform Terraform Module

Executive Summary

This Terraform module provides a unified interface for managing the complete IAM lifecycle on Google Cloud Platform. It handles service account creation and key management, custom IAM roles at both project and organization levels, IAM member bindings across project/folder/organization scopes with conditional access, Workload Identity Federation with OIDC and AWS providers, service account impersonation, and organization policy enforcement. All resources support dynamic creation via for_each with comprehensive input validation.

Component Breakdown

Service Accounts & Keys

Creates service accounts via google_service_account with configurable display name, description, and disabled state. Keys generated with RSA-2048 algorithm. Project ID defaults from module variable but can be overridden per account.

Custom IAM Roles

Supports both google_project_iam_custom_role and google_organization_iam_custom_role. Roles accept lists of permissions with lifecycle stage (ALPHA through GA). Validated against allowed stage values.

IAM Bindings (3 Scopes)

Uses google_project_iam_member, google_folder_iam_member, and google_organization_iam_member. All support multiple members per binding and optional IAM conditions with CEL expressions.

Workload Identity Federation

Creates google_iam_workload_identity_pool with nested providers supporting OIDC (with issuer URI, audiences, JWKS) and AWS (account ID). Attribute mappings and conditions for token exchange.

Service Account Impersonation

Grants roles/iam.serviceAccountTokenCreator to specified members with optional conditions. Enables cross-service-account delegation.

Organization Policies

Applies google_project_organization_policy constraints with boolean enforcement or list-based allow/deny values.

Industry Adaptation

Enterprise

Multi-level IAM bindings (project/folder/org). Org policies for governance. Workload Identity for CI/CD pipelines.

Regulated Industries

Conditional access with time-bound permissions. Org policies to restrict regions and services.

Multi-Cloud

WIF with AWS providers for cross-cloud workloads. OIDC for GitHub Actions, GitLab, Azure DevOps.

Startups

Simple service accounts with project IAM bindings. Grow into custom roles and org policies as needed.

Production Readiness Checklist

  • Use Workload Identity Federation instead of exported keys
  • Apply least-privilege IAM bindings with specific roles
  • Enable conditional access for sensitive permissions
  • Enforce org policies for security baselines
  • Use custom roles when predefined roles are too broad
  • Audit impersonation bindings regularly
  • Validate all inputs with regex constraints
  • Scope IAM at the narrowest level (project > folder > org)
